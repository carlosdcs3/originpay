## 7. Síntese dos bloqueios corrigidos

### Bloqueio 1 — documentos não encontrados

Causa:

- A sessão estava em `core`.
- Os documentos solicitados estavam em `../docs`, ou seja, no `docs` da raiz superior.

Correção operacional:

- Para próximas fases, quando o usuário referenciar os documentos de produção/auditoria históricos, usar explicitamente:

```text
E:\projetos\DigiKash v1.0.5\DigiKash v1.0.5\docs
```

- Para documentos gerados durante as fases executadas a partir da aplicação Laravel, o caminho relativo atual `docs/...` aponta para:

```text
E:\projetos\DigiKash v1.0.5\DigiKash v1.0.5\core\docs
```

### Bloqueio 2 — PHP não disponível no PATH

Causa:

- `php` não está no PATH do shell atual.
- O Laragon não está em `C:\laragon`, mas sim em `E:\laragon`.

Correção operacional:

- Usar PHP absoluto:

```text
E:\laragon\bin\php\php-8.3.30-Win32-vs16-x64\php.exe
```

- Usar Composer absoluto via PHAR quando necessário:

```text
E:\laragon\bin\composer\composer.phar
```
